SEMÂNTICA HTML

O que significa semântica?

O texto em html é formatado através das tags. Além da formatação visual, as tags também carregam um sentido que, na hora de ser interpretado pelos motores de busca e leitura, ganham relevância e destaque.
Por exemplo, qual a diferença entre strong e bold?
Apesar de os dois estarem em negrito, o motor de busca assimilará a relevância da informação de forma distinta.

Tag font

Esta tag é utilizada para formatar letras. O CSS apresenta mais modularidade e possiblidades para uso de fontes, mas entender o html ajuda a compreender os futuros atributos passados por css.

Este é um exemplo

<h2><font color="#ee0000" face="'Trebuchet MS',Tahoma"> Este </font>é um exemplo</h2>

  • atributo color: É possível selecionar uma cor através de código RGB, CMYK e alguns navegadores suportam o nome da cor.

  • atributo face:Seleciona a fonte indicada. O navegador assume que o usuário que acessa o site possui a fonte instalada. Caso não haja, o navegador usará a fonte padrão.

    Para garantir que as fontes estarão adequadas, podemos passar mais de uma fonte pelo parâmetro.

Tag div e span

Estas tags servem como "tags coringas", pois sua função é encapsular uma sessão da página para assim organizá-la de forma modular.
Visualmente, essas tags não alteram a aparência do site, sendo necessário o uso de CSS para tal. Essa é a vantagem de se utilizar a DIV, já que podemos definir um estilo para a sessão do site, afetando apenas os elementos filhos.

  • DIV:Geralmente usado como demarcador de sessão. Por padrão vem com o atributo display: block, assim ocupando a linha horizontal da página.
  • SPAN: Usado como tag auxiliar para elementos que estejam dentro de uma div, pois por possuir o display padrão não afeta na posição dos outros elementos da página

Tag embed

Hoje a tag embed possui uma função obsoleta. Mas seu entendimento nos ajuda a compreender a evolução tecnológica da internet e as minúcias do funcionamento de uma página.

Nos anos 90 e 2000, as páginas eram muito mais humildes comparadas ao que possuímos hoje: a grande maioria das páginas demoravam para carregar simples imagens, vídeos e até parágrafos de texto. Tanto que, em sua maioria, o conteúdo se limitava a apenas elementos HTML, estilos simples de CSS e interações simples em Javascript.

Uma página HTML é carregada de cima para baixo simultaneamente, e, portanto, caso uma parte do conteúdo demorasse mais para ser carregada, o usuário só teria acesso ao resto da página assim que esse elemento terminasse de ser baixado.

Para contornar este problema, os antigos desenvolvedores utilizam a técnica dos frames, que consistia em carregar diferentes sessões da página como se fossem diferentes páginas. Por mais que ainda demorasse para carregar a página por completo, ao menos o usuário conseguiria acessar os conteúdos mais leves, como textos, mais rapidamente.

Porém, se uma div possuísse uma parágrafo seguido de um vídeo, por mais que não necessariamente fosse o frame que mais demorasse para carregar, o usuário demoraria muito tempo para ler o texto, já que ele só teria acesso ao conteúdo quando o vídeo terminasse de carregar.

Dessa problemática surge a utilidade da tag Embed. Esta tag carrega o conteúdo independente do carregamento do frame, permitindo que conteúdos mais leves fossem disponibilizados antes que o conteúdo dentro da tag terminasse de carregar.

Por questões de compatibilidade, ainda hoje é possível utilizar a tag embed nas páginas. Contudo, com a tecnologia atual do HTML 5, que disponibiliza tags com semântica mais relevante e clara, tornou o uso do embed obsoleto em sites modernos.

A tag embed pode inserir qualquer conteúdo multimídia à página, desde tecnologias obsoletas como animações em Flash Player até vídeos e imagens. Porém, hoje em dia existem tags específicas para cada caso, como img para imagens e vídeo para vídeos. Além de essas tags fornecerem conteúdo multimidia, também tornam a semântica do site mais otimizada, pois o motor de busca ou leitura sabe que aquele elemento se trata de uma imagem ou vídeo, enquanto embed pode ser qualquer conteúdo multimídia.
